Sync Web API

Sync utilizes Spotify’s Web API to get information about public playlists that you and your friends listen to. Sync then creates a synced playlist that contains overlapping music from you and your friends’ playlists. As an added feature, Sync will email the synced playlist to you and your friends so you can save your sync to Spotify.

Note, the api prompts users to input spotify usernames and playlist ids. These can be retrieved from www.open.spotify.com and navigating to a users profile or a playlist page. You can then retrieve the username of the user or id of the playlist by copying very end of the end point. Ex. https://open.spotify.com/playlist/37i9dQZF1DWWBHeXOYZf74 is a page for the POLLEN playlist and 37i9dQZF1DWWBHeXOYZf74 is the playlist id. '

post /api/playlists/{username}/
(apiPlaylistsUsernamePost)
Adds a public spotify playlist to a users profile.

Path parameters

username (required)
Path Parameter — Spotify username of a saved user.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body map (required)
Body Parameter —
example: { "name" : "name of spotify playlist ex. POLLEN", "playlist" : "spotify playlist id ex. 37i9dQZF1DWWBHeXOYZf74" }

Return type

map[String, String]

Example data

Content-Type: application/json
{
  "success" : true,
  "data" : {
    "username" : "username",
    "email" : "email",
    "playlists" : [ "", "", "...", "" ],
    "syncs" : [ ]
  }
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Successful retrieval of spotiy user info.

404

Users profile not found.

post /api/syncs/
(apiSyncsPost)
Creates a synced between two users containing overlapping songs from their respective playlists.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body map (required)
Body Parameter —
example: { "user_one" : "spotify username ex. tedi.m52", "user_two:" : "spotify username ex. 6wkaxax29ty3f2z2eyappmk13" }

Return type

map[String, String]

Example data

Content-Type: application/json
{
  "success" : true,
  "data" : {
    "users" : [ {
      "username" : "tedi.m52"
    }, {
      "username" : "6wkaxax29ty3f2z2eyappmk13"
    } ],
    "synced_playlist" : [ "", "", "..." ]
  }
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Successful creation of synced playlist for user_one and user_two.

404

One of the users profile were not found.
